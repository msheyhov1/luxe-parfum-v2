# LUXE Парфюмерия — v2 (новый дизайн, распакованный проект)

Вторая версия сайта — новый дизайн заказчика, **распакованный** из офлайн-бандла в
редактируемые файлы.

## ⚠️ Как открывать
Открывать **через локальный сервер**, не двойным кликом (рантайм подгружает компонент
карточки через `fetch`, а это не работает по `file://`):

```bash
# из корня проекта (где уже запущен сервер):
#   http://localhost:8000/v2/index.html
# или поднять свой:
cd "путь/к/dima pardum/v2" && python3 -m http.server 8001
#   затем http://localhost:8001/index.html
```

## Что умеет сайт (доработан до полноценного)
Все окна и страницы рабочие, переходы и состояния реализованы целиком:

- **Главная** — hero-слайдер (стрелки/точки/свайп/автоплей), блок преимуществ,
  **карусель «Интересные ароматы» (рекомендации) со стрелками**, каталог с фильтрами
  (семейство, бренд, пол, сортировка, поиск), философия дома.
- **Каталог/карточки** — наведение раскрывает пирамиду нот, «в избранное», «в корзину»
  прямо с карточки, пустое состояние со сбросом фильтров. **Если товар уже в корзине,
  на карточке вместо «+» появляется счётчик с кнопками − / + (количество в корзине).**

### Админ-панель (доработка)
Ссылка **«· Админ»** в футере (рядом с копирайтом) → вход по паролю → редактор всего
контента сайта. Пароль по умолчанию **`luxe2026`** — меняется в `index.html`
(константа `this.ADMIN_PASS` в конструкторе, ≈ строка 655).

Что можно редактировать (вкладки):
- **Товары** — добавить/удалить, изменить бренд, название, семейство, пол, бейдж,
  описание, свечение (RGB), ноты (верх/сердце/база), объёмы и цены (+ выбор основного
  объёма для цены на карточке). Фото — путь (`photos/…`) **или загрузка файла** прямо
  из панели.
- **Слайдер** — добавить/удалить hero-слайды, все тексты, свечение, фото/загрузка.
- **Отзывы** — добавить/удалить, имя, дата, рейтинг ★, текст.
- **Оформление** — акцентный цвет сайта.

Изменения сохраняются автоматически в `localStorage` этого браузера (ключ
`luxe_admin_v2`) и сразу видны на сайте. Чтобы они стали **постоянными для всех
посетителей**, нажмите **«Экспорт JSON»** и передайте файл разработчику (данные
вставляются в `this.products` / `this.heroSlides` / `this.reviews` в `index.html`).
Кнопка **«Сбросить всё к исходному»** (вкладка «Оформление») очищает правки.
> Загрузка фото хранит картинку как data-URL в `localStorage` — для нескольких крупных
> фото возможно превышение лимита (~5 МБ); тогда правки действуют до перезагрузки.
> Надёжнее класть файлы в `photos/` и указывать путь.
- **Страница товара** — галерея-миниатюры, выбор объёма, количество, добавление в корзину,
  избранное, пирамида аромата, аккордеоны, **рейтинг ★ и блок отзывов**, похожие ароматы.
- **Корзина → Оформление → Готово** — изменение количества, удаление, промокод `LUXE`
  (−10%), форма доставки/оплаты, подтверждение с номером заказа.
- **Личный кабинет** (иконка пользователя в шапке) — вход и регистрация (вкладки,
  валидация), профиль с аватаром-инициалом и **историей заказов**, выход. Аккаунты и
  сессия хранятся в `localStorage` (`luxe_users`, `luxe_session`).
  > Авторизация демонстрационная: пароли лежат в `localStorage` в открытом виде.
- **Заказы** — при оформлении залогиненным пользователем заказ пишется в его историю;
  форма оформления **предзаполняется** данными из профиля.
- **Инфо-страницы** (ссылки в футере «Клиентам») — О нас, Доставка и оплата, Возврат и
  обмен, Подлинность, Подарочные карты, Контакты (с карточками телефон/e-mail/адрес/часы).
  Контент в `this.infoData` в `index.html`.
- **Поиск** работает с любой страницы: переключает на главную, фильтрует каталог и
  **прокручивает к результатам** (раньше казалось, что «не работает» — результат уходил
  под экран). На мобильной поиск живёт в бургер-меню (кнопка «Найти» закрывает меню и
  ведёт к каталогу).
- **Рассылка в футере** — с валидацией e-mail и сообщением «Спасибо за подписку».

### Мобильная версия
Полноценная адаптивная вёрстка (брейкпоинты 880px / 560px):
- **Бургер-меню** — на мобильной верхняя навигация и поиск прячутся, появляется кнопка
  «☰». Открывает полноэкранную панель: поиск + «Найти», все разделы каталога, кнопки
  «Кабинет» и «Корзина». Иконки аккаунт/избранное/корзина остаются в шапке.
- Все двухколоночные раскладки (товар, корзина, оформление, футер, пирамида) сворачиваются
  в одну колонку; убраны горизонтальные переполнения; уменьшены отступы.
- **Каталог на мобильной — в 2 карточки в ряд** (а не в одну): правило
  `[style*="minmax(244px, 1fr)"]{grid-template-columns:repeat(2,1fr)}` в `@media`
  (880px и 560px). Карусель рекомендаций листается стрелками и свайпом.
- Реализация — в `<style>` внутри `<helmet>` (`@media`). Важно: рантайм сериализует
  inline-стили как `prop: value;` (с пробелом после двоеточия) — **селекторы
  `[style*="…"]` должны использовать именно эту форму с пробелом**, иначе не сработают.
  Классы `lx-nav` / `lx-search` / `lx-icons` / `lx-burger` / `lx-mnav` управляют шапкой и
  панелью.

Корзина и избранное переживают перезагрузку (`luxe_cart`, `luxe_wish`).

## Как это устроено
Это страница на рантайме `app.js` (custom-элементы `<x-dc>`, шаблоны `{{ ... }}`).
Вся **редактируемая часть** — в `index.html`, внутри `<script type="text/x-dc">`:
- массив товаров `this.products = [ ... ]`;
- данные отзывов `this.reviews`, инфо-страниц `this.infoData`, ссылок футера `this.clientLinks`;
- логика отображения `renderVals()` / `cardVM()` (цены, бейджи, **картинки**, аккаунт `av`, инфо `iv`).

## 🖼 Где вставляются фото (главное)
В оригинальном дизайне **фото нет** — вместо них цветной градиент-«свечение» (`glow`)
и подписи-плейсхолдеры (`ФОТО · BRAND`, `HERO · FULL-BLEED PRODUCT PHOTO`).

Картинка товара — это CSS-фон `imgBg`. Он применяется в 4 местах: карточка каталога,
большое фото на странице товара, миниатюры, строка корзины.

**Чтобы поставить реальное фото** достаточно добавить товару поле `photo` — добавлен
помощник, который сам подставит фото вместо градиента везде:

```js
// в this.products у нужного товара:
{ id:'creed', brand:'CREED', name:'Millésime Impérial', …, glow:'201,161,95',
  photo:'photos/creed.jpg',   // ← вот так. без photo — остаётся градиент
  … }
```

Как это работает (уже встроено):
```js
photoBg(p, grad) { return p.photo ? ('#0d0d0f center/cover no-repeat url("' + p.photo + '")') : grad; }
// и во всех imgBg:  imgBg: this.photoBg(p, '<градиент по умолчанию>')
```

### Добавить фото своему товару — 2 шага
1. Положи файл в `photos/`, например `photos/tomford.jpg`.
2. В `index.html` найди нужный товар в `this.products` и допиши `photo:'photos/tomford.jpg',`.

Готово — фото появится на карточке, на странице товара, в миниатюрах и корзине.
Подпись `ФОТО · …` при наличии фото скрывается автоматически.

> **Фото уже подставлены везде:** все 8 товаров и 3 hero-слайда имеют `photo:` (файлы в
> `photos/`). Чтобы заменить — просто положи свой файл с тем же именем или поменяй путь
> в `photo:`.

### Hero-баннер
Слайды hero (`this.heroSlides`, ≈ строка 589) тоже получили `photo:` (`photos/hero-1..3.jpg`);
фон собирается тем же помощником (`heroBg: this.photoBg(slide, …)`). Подпись-плейсхолдер
скрывается при наличии фото.

## Листание свайпом (не только кнопками)
В конце `index.html` добавлен небольшой делегированный скрипт-«мост»: hero можно
**перетаскивать мышью, свайпать пальцем и листать трекпадом** (горизонтально), а не только
жать стрелки. Он находит карусель по кнопкам `aria-label="Назад/Вперёд"` и «нажимает»
нужную на свайпе — поэтому переживает перерисовку рантайма. Стрелки и точки продолжают
работать. (Карточки товара в v2 — одиночное изображение, без галереи, поэтому свайпать там
нечего.)

## Соотношение с v1
- **v1** (корень) — отрефакторенная версия на чистых HTML/CSS/JS (своя кодовая база).
- **v2** (эта папка) — новый дизайн на рантайме дизайн-инструмента. Это **разные** проекты;
  общего кода нет.
